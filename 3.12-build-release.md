# 3.12 Build & Release

> **Mục tiêu**: Hiểu quy trình build, deploy và release ứng dụng React Native (Expo) lên App Store (iOS) và Google Play (Android) — từ config, EAS Build, OTA updates đến submission.

---

## 📖 Tổng Quan

### Quy Trình Build & Release

```
┌─ Development ──────────────────────────────────────────────┐
│                                                             │
│  Code → Config app.json → EAS Build → Testing              │
│                     │                     │                 │
│                     ▼                     ▼                 │
│              Development Build      Preview Build           │
│             (dev-client, debug)   (internal testing)        │
│                                         │                  │
│                                         ▼                  │
│                                  Production Build           │
│                                    │           │           │
│                                    ▼           ▼           │
│                              App Store    Google Play       │
│                              Connect       Console          │
│                                    │           │           │
│                                    ▼           ▼           │
│                              TestFlight   Internal Test    │
│                                    │           │           │
│                                    ▼           ▼           │
│                               Production Release           │
│                                                             │
│  ──── OTA Updates (expo-updates) ────────────────           │
│  Minor fixes → Publish update → Users get instantly         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Build Types

| Type            | Mục đích                          | Distribution            |
| --------------- | --------------------------------- | ----------------------- |
| **Development** | Dev & debug, có dev tools         | Device cài trực tiếp    |
| **Preview**     | Internal testing, không dev tools | Internal testers        |
| **Production**  | Release cho users                 | App Store / Google Play |

### 1.2 Dynamic Config (app.config.js)

Dùng khi cần đọc environment variables hoặc logic:

```js
// app.config.js
import "dotenv/config";

const IS_PROD = process.env.APP_ENV === "production";
const IS_PREVIEW = process.env.APP_ENV === "preview";

const getAppName = () => {
  if (IS_PROD) return "MyApp";
  if (IS_PREVIEW) return "MyApp (Preview)";
  return "MyApp (Dev)";
};

const getBundleId = () => {
  if (IS_PROD) return "com.yourcompany.myapp";
  if (IS_PREVIEW) return "com.yourcompany.myapp.preview";
  return "com.yourcompany.myapp.dev";
};

export default ({ config }) => ({
  ...config,
  name: getAppName(),
  ios: {
    ...config.ios,
    bundleIdentifier: getBundleId(),
  },
  android: {
    ...config.android,
    package: getBundleId(),
  },
  extra: {
    ...config.extra,
    apiUrl: process.env.API_URL || "http://localhost:3000",
    appEnv: process.env.APP_ENV || "development",
  },
});
```

### 1.3 Environment Variables

```bash
# .env.development
API_URL=http://localhost:3000
APP_ENV=development

# .env.preview
API_URL=https://staging-api.yourcompany.com
APP_ENV=preview

# .env.production
API_URL=https://api.yourcompany.com
APP_ENV=production
```

Đọc biến trong code:

```tsx
import Constants from "expo-constants";

const API_URL = Constants.expoConfig?.extra?.apiUrl;
const APP_ENV = Constants.expoConfig?.extra?.appEnv;

console.log("Environment:", APP_ENV);
console.log("API URL:", API_URL);
```

## 2. EAS Build Setup

### 2.1 Cài Đặt EAS CLI

```bash
# Cài global
npm install -g eas-cli

# Login vào Expo account
eas login

# Init EAS trong project
eas build:configure
```

### 2.2 eas.json — Build Profiles

```json
{
  "cli": {
    "version": ">= 12.0.0",
    "appVersionSource": "remote"
  },
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal",
      "ios": {
        "simulator": true
      },
      "env": {
        "APP_ENV": "development",
        "API_URL": "http://localhost:3000"
      }
    },
    "preview": {
      "distribution": "internal",
      "autoIncrement": true,
      "env": {
        "APP_ENV": "preview",
        "API_URL": "https://staging-api.yourcompany.com"
      },
      "channel": "preview"
    },
    "production": {
      "autoIncrement": true,
      "env": {
        "APP_ENV": "production",
        "API_URL": "https://api.yourcompany.com"
      },
      "channel": "production"
    }
  },
  "submit": {
    "production": {
      "ios": {
        "appleId": "your@email.com",
        "ascAppId": "1234567890",
        "appleTeamId": "XXXXXXXXXX"
      },
      "android": {
        "serviceAccountKeyPath": "./google-services.json",
        "track": "internal"
      }
    }
  }
}
```

### 2.3 Build Commands

```bash
# === Development Build ===
# iOS Simulator
eas build --profile development --platform ios

# Android (APK để cài trực tiếp)
eas build --profile development --platform android

# === Preview Build (Internal Testing) ===
eas build --profile preview --platform all

# === Production Build ===
# iOS (IPA cho App Store)
eas build --profile production --platform ios

# Android (AAB cho Google Play)
eas build --profile production --platform android

# Cả hai platforms
eas build --profile production --platform all
```

## 3. Versioning

### 3.1 Quy Tắc Version

```
version: "1.2.3"
         │ │ │
         │ │ └─ PATCH — Bug fixes, minor changes
         │ └─── MINOR — New features (backward compatible)
         └───── MAJOR — Breaking changes

buildNumber (iOS) / versionCode (Android):
  → Số tăng dần mỗi lần build mới upload lên store
  → EAS auto-increment nếu bật "autoIncrement": true
```

### 3.2 Auto Version Management

```json
// eas.json
{
  "cli": {
    "appVersionSource": "remote" // EAS quản lý version
  },
  "build": {
    "production": {
      "autoIncrement": true // Tự tăng buildNumber/versionCode
    }
  }
}
```

```bash
# Cập nhật version thủ công
eas build:version:set --platform ios --build-number 10
eas build:version:set --platform android --version-code 10
```

## 4. OTA Updates (Over-The-Air)

> OTA cho phép update JS bundle **không cần** qua App Store / Google Play.  
> Chỉ áp dụng cho JS/assets changes, **không** thể thay đổi native code.

### 4.1 Setup

```bash
npx expo install expo-updates
```

### 4.2 Cấu Hình

```json
// app.json
{
  "expo": {
    "updates": {
      "enabled": true,
      "fallbackToCacheTimeout": 0,
      "url": "https://u.expo.dev/your-project-id"
    },
    "runtimeVersion": {
      "policy": "appVersion"
    }
  }
}
```

### 4.3 Publish Update

```bash
# Publish OTA update cho channel "production"
eas update --branch production --message "Fix login bug"

# Publish cho channel "preview"
eas update --branch preview --message "Add new feature"

# Xem các updates
eas update:list
```

### 4.4 Kiểm Tra Update Trong App

```tsx
import * as Updates from "expo-updates";
import { Alert } from "react-native";

// Check for updates manually
async function checkForUpdate() {
  try {
    const update = await Updates.checkForUpdateAsync();

    if (update.isAvailable) {
      Alert.alert(
        "Update Available",
        "A new version is available. Would you like to update?",
        [
          { text: "Later", style: "cancel" },
          {
            text: "Update Now",
            onPress: async () => {
              await Updates.fetchUpdateAsync();
              await Updates.reloadAsync(); // Restart app với update mới
            },
          },
        ],
      );
    }
  } catch (error) {
    console.error("Error checking for updates:", error);
  }
}

// Chạy khi app mở
// Hoặc gọi ở useEffect trong root component
```

### 4.5 Giới Hạn OTA

| Có thể OTA ✅          | Không thể OTA ❌                 |
| ---------------------- | -------------------------------- |
| JS/TS code changes     | Native module mới (camera, etc.) |
| Style changes          | app.json config changes          |
| Image/asset updates    | Splash screen, app icon          |
| New screens/components | Permissions mới                  |
| API endpoint changes   | expo-updates config              |

## 5. App Store Submission

### 5.1 iOS — App Store Connect

**Chuẩn bị:**

```
assets/
├── icon.png                  (1024x1024, no alpha)
├── splash.png                (1284x2778)
├── screenshots/
│   ├── iphone-6.5/           (1284x2778) — iPhone 14 Pro Max
│   │   ├── screenshot-1.png
│   │   ├── screenshot-2.png
│   │   └── screenshot-3.png
│   ├── iphone-5.5/           (1242x2208) — iPhone 8 Plus
│   └── ipad-12.9/            (2048x2732) — iPad Pro
└── store/
    ├── description.txt        (4000 chars max)
    ├── keywords.txt           (100 chars, comma-separated)
    ├── privacy-policy-url.txt
    └── support-url.txt
```

**Submit:**

```bash
# Build production
eas build --profile production --platform ios

# Submit lên App Store Connect
eas submit --platform ios

# Hoặc build + submit cùng lúc
eas build --profile production --platform ios --auto-submit
```

**Sau khi submit:**

1. Vào [App Store Connect](https://appstoreconnect.apple.com/)
2. Chọn app → "App Store" tab
3. Fill: Description, Keywords, Screenshots, Privacy Policy URL
4. Select build vừa upload
5. Submit for Review
6. Chờ Apple review (thường 1-3 ngày)

### 5.2 Android — Google Play Console

**Chuẩn bị:**

```
assets/
├── icon.png                  (512x512)
├── feature-graphic.png       (1024x500) — Banner
├── screenshots/
│   ├── phone/                (min 2, max 8)
│   │   ├── screenshot-1.png  (min 320px, max 3840px)
│   │   └── screenshot-2.png
│   └── tablet/               (optional)
└── store/
    ├── short-description.txt  (80 chars max)
    ├── full-description.txt   (4000 chars max)
    └── privacy-policy-url.txt
```

**Submit:**

```bash
# Build production (AAB format)
eas build --profile production --platform android

# Submit lên Google Play
eas submit --platform android

# Build + submit
eas build --profile production --platform android --auto-submit
```

**Google Play tracks:**

| Track            | Mục đích                    |
| ---------------- | --------------------------- |
| Internal testing | Team nội bộ (< 100 testers) |
| Closed testing   | Beta testers (invite)       |
| Open testing     | Public beta                 |
| Production       | Public release              |

## 6. Signing & Certificates

### 6.1 iOS Signing

```bash
# EAS tự quản lý certificates (recommended)
# Khi build lần đầu, EAS sẽ tạo:
# - Distribution Certificate
# - Provisioning Profile

# Xem credentials
eas credentials --platform ios

# Reset credentials nếu cần
eas credentials --platform ios
# → Chọn "Remove" → Tạo lại ở build tiếp theo
```

### 6.2 Android Signing

```bash
# EAS tự tạo keystore (recommended)
# Khi build lần đầu, EAS tạo:
# - Upload key (cho Google Play)
# - Keystore

# Xem credentials
eas credentials --platform android

# ⚠️ QUAN TRỌNG: Backup keystore!
# Nếu mất keystore → không thể update app trên Play Store
```

## 7. CI/CD với EAS

### 7.1 GitHub Actions

```yaml
# .github/workflows/eas-build.yml
name: EAS Build & Submit

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  # PR → Preview build
  preview:
    if: github.event_name == 'pull_request'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install dependencies
        run: npm ci

      - name: Setup EAS
        uses: expo/expo-github-action@v8
        with:
          eas-version: latest
          token: ${{ secrets.EXPO_TOKEN }}

      - name: Build preview
        run: eas build --profile preview --platform all --non-interactive

  # Main branch → Production build & submit
  production:
    if: github.ref == 'refs/heads/main' && github.event_name == 'push'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install dependencies
        run: npm ci

      - name: Setup EAS
        uses: expo/expo-github-action@v8
        with:
          eas-version: latest
          token: ${{ secrets.EXPO_TOKEN }}

      - name: Build & Submit
        run: eas build --profile production --platform all --auto-submit --non-interactive

  # OTA update for non-native changes
  update:
    if: github.ref == 'refs/heads/main' && github.event_name == 'push'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install dependencies
        run: npm ci

      - name: Setup EAS
        uses: expo/expo-github-action@v8
        with:
          eas-version: latest
          token: ${{ secrets.EXPO_TOKEN }}

      - name: Publish OTA Update
        run: eas update --branch production --message "${{ github.event.head_commit.message }}" --non-interactive
```

### 7.2 Expo Token

```bash
# Tạo token tại https://expo.dev/accounts/[account]/settings/access-tokens
# Thêm vào GitHub Secrets:
# Settings → Secrets → Actions → New: EXPO_TOKEN
```

## 8. Release Checklist

### Pre-Release

```markdown
## Before submitting to stores:

### Code Quality

- [ ] No console.log/warn in production
- [ ] Error boundaries for crash handling
- [ ] All API endpoints point to production
- [ ] Environment variables set correctly

### Performance

- [ ] App loads < 3 seconds
- [ ] No memory leaks
- [ ] Images optimized
- [ ] Bundle size acceptable

### Testing

- [ ] Manual testing on iOS device
- [ ] Manual testing on Android device
- [ ] All critical flows work (login, main features, payment)
- [ ] Edge cases handled (no network, empty states)
- [ ] Deep links work correctly

### Assets

- [ ] App icon (iOS: 1024x1024, Android: 512x512)
- [ ] Splash screen
- [ ] Screenshots for all required device sizes
- [ ] App Store description + keywords
- [ ] Privacy policy URL
- [ ] Support URL

### Configuration

- [ ] Version number updated
- [ ] Bundle identifier correct
- [ ] Permissions declared & descriptions filled
- [ ] Signing certificates valid
```

### Post-Release

```markdown
## After release:

- [ ] Verify app available in stores
- [ ] Test production download & install
- [ ] Monitor crash reports (Sentry/Crashlytics)
- [ ] Check analytics events
- [ ] Announce release to team
- [ ] Tag release in git: git tag v1.0.0
```

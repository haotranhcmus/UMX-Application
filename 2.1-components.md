# 2.1 Components

> **Mục tiêu**: Hiểu component-based architecture, tạo và tổ chức components trong React Native.

---

## 📖 Tổng Quan

Trong React Native, mọi thứ đều là **component**. Một component là một khối UI độc lập, có thể tái sử dụng. Giống như LEGO — bạn xây dựng UI phức tạp bằng cách kết hợp các component nhỏ lại với nhau.

**Component-based architecture** mang lại:

- ♻️ **Reusability** — Viết 1 lần, dùng nhiều nơi
- 🧩 **Modularity** — Dễ chia nhỏ, dễ quản lý
- 🧪 **Testability** — Test từng component riêng lẻ
- 🛠 **Maintainability** — Sửa 1 chỗ, cập nhật khắp nơi

---

## 1. Function Components

### 1.1 Tạo Function Component

Function component là một **hàm JavaScript** trả về JSX (giao diện). Đây là cách viết component phổ biến nhất hiện nay.

```tsx
// components/Greeting.tsx
import React from "react";
import { View, Text, StyleSheet } from "react-native";

// ✅ Function Component cơ bản
const Greeting = () => {
  return (
    <View style={styles.container}>
      <Text style={styles.text}>Xin chào React Native! 👋</Text>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    padding: 16,
    backgroundColor: "#f0f0f0",
    borderRadius: 8,
  },
  text: {
    fontSize: 18,
    fontWeight: "bold",
    color: "#333",
  },
});

export default Greeting;
```

### 1.2 Return JSX

JSX (JavaScript XML) là cú pháp giống HTML để mô tả UI. Trong React Native, ta dùng các component như `View`, `Text` thay vì `div`, `p`.

```tsx
// ✅ Mỗi component chỉ return 1 root element
const MyComponent = () => {
  return (
    <View>
      <Text>Dòng 1</Text>
      <Text>Dòng 2</Text>
    </View>
  );
};

// ❌ SAI — return nhiều root elements
const BadComponent = () => {
  return (
    <Text>Dòng 1</Text>
    <Text>Dòng 2</Text>  // Lỗi!
  );
};
```

### 1.3 Component Naming Convention (PascalCase)

**Quy tắc bắt buộc**: Tên component PHẢI viết **PascalCase** (chữ cái đầu viết hoa).

```tsx
// ✅ ĐÚNG — PascalCase
const UserProfile = () => { ... };
const HeaderNavigation = () => { ... };
const TodoListItem = () => { ... };

// ❌ SAI — camelCase hoặc lowercase
const userProfile = () => { ... };   // React sẽ hiểu là HTML tag
const header_navigation = () => { ... };
```

**Tại sao?** React phân biệt component (`<UserProfile />`) với HTML/native element (`<view>`) dựa trên chữ cái đầu tiên.

## 2. JSX Chi Tiết

### 2.1 Cú pháp JSX

JSX là "syntactic sugar" — nó được biên dịch thành các lệnh `React.createElement()`.

```tsx
// Viết JSX:
<View style={styles.box}>
  <Text>Hello</Text>
</View>;

// Tương đương JavaScript thuần:
React.createElement(
  View,
  { style: styles.box },
  React.createElement(Text, null, "Hello"),
);
```

**Các quy tắc JSX cần nhớ:**

```tsx
import React from "react";
import { View, Text, Image } from "react-native";

const JSXRules = () => {
  return (
    <View>
      {/* 1. Dùng className → style trong RN */}
      <View style={{ padding: 10 }} />

      {/* 2. Self-closing tags */}
      <Image source={{ uri: "https://example.com/img.png" }} />

      {/* 3. camelCase cho props */}
      <Text style={{ fontSize: 16, fontWeight: "bold" }}>Text</Text>

      {/* 4. Boolean props */}
      <Text selectable={true}>Selectable text</Text>
      <Text selectable>Shorthand — cũng là true</Text>
    </View>
  );
};
```

### 2.2 Expressions trong JSX: `{variable}`

Dùng `{}` để nhúng bất kỳ JavaScript expression nào vào JSX.

```tsx
import React from "react";
import { View, Text, StyleSheet } from "react-native";

const ExpressionsDemo = () => {
  const name = "Hào";
  const age = 22;
  const hobbies = ["Code", "Game", "Coffee"];
  const today = new Date().toLocaleDateString("vi-VN");

  return (
    <View style={styles.container}>
      {/* Biến */}
      <Text>Xin chào, {name}!</Text>

      {/* Biểu thức tính toán */}
      <Text>
        Tuổi: {age} — Năm sinh: {2026 - age}
      </Text>

      {/* Gọi method */}
      <Text>Tên viết hoa: {name.toUpperCase()}</Text>

      {/* Template literal */}
      <Text>{`Hôm nay: ${today}`}</Text>

      {/* Array join */}
      <Text>Sở thích: {hobbies.join(", ")}</Text>
    </View>
  );
};

const styles = StyleSheet.create({
  container: { padding: 16, gap: 8 },
});

export default ExpressionsDemo;
```

### 2.3 Conditional Rendering

Hiển thị UI khác nhau dựa trên điều kiện.

```tsx
import React from "react";
import { View, Text, StyleSheet } from "react-native";

interface ConditionalDemoProps {
  isLoggedIn: boolean;
  username?: string;
  notificationCount: number;
  role: "admin" | "user" | "guest";
}

const ConditionalDemo = ({
  isLoggedIn,
  username,
  notificationCount,
  role,
}: ConditionalDemoProps) => {
  return (
    <View style={styles.container}>
      {/* 1️⃣ Toán tử && — render nếu điều kiện đúng */}
      {isLoggedIn && <Text>Chào mừng trở lại!</Text>}

      {/* ⚠️ Cẩn thận với số 0 — dùng Boolean() hoặc !! */}
      {notificationCount > 0 && (
        <Text>Bạn có {notificationCount} thông báo</Text>
      )}

      {/* 2️⃣ Ternary — if/else inline */}
      <Text>{isLoggedIn ? `Xin chào, ${username}` : "Vui lòng đăng nhập"}</Text>

      {/* 3️⃣ Ternary với component */}
      {isLoggedIn ? (
        <View style={styles.loggedIn}>
          <Text>Dashboard</Text>
        </View>
      ) : (
        <View style={styles.loggedOut}>
          <Text>Login Form</Text>
        </View>
      )}

      {/* 4️⃣ Multiple conditions — dùng hàm hoặc early return */}
      {role === "admin" && <Text>🛡 Admin Panel</Text>}
      {role === "user" && <Text>👤 User Dashboard</Text>}
      {role === "guest" && <Text>👁 View Only</Text>}
    </View>
  );
};

const styles = StyleSheet.create({
  container: { padding: 16, gap: 8 },
  loggedIn: { backgroundColor: "#d4edda", padding: 12, borderRadius: 8 },
  loggedOut: { backgroundColor: "#f8d7da", padding: 12, borderRadius: 8 },
});

export default ConditionalDemo;
```

### 2.4 List Rendering với `map()` và `key`

Render danh sách từ array data.

```tsx
import React from "react";
import { View, Text, StyleSheet } from "react-native";

interface User {
  id: string;
  name: string;
  email: string;
}

const ListDemo = () => {
  const users: User[] = [
    { id: "1", name: "Hào", email: "hao@example.com" },
    { id: "2", name: "Minh", email: "minh@example.com" },
    { id: "3", name: "Lan", email: "lan@example.com" },
  ];

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Danh sách Users:</Text>

      {/* ✅ Luôn có key duy nhất — dùng id, KHÔNG dùng index */}
      {users.map((user) => (
        <View key={user.id} style={styles.userCard}>
          <Text style={styles.name}>{user.name}</Text>
          <Text style={styles.email}>{user.email}</Text>
        </View>
      ))}
    </View>
  );
};

// ❌ SAI — Dùng index làm key
// {users.map((user, index) => (
//   <View key={index}>...</View>  // Gây bug khi list thay đổi!
// ))}

const styles = StyleSheet.create({
  container: { padding: 16 },
  title: { fontSize: 20, fontWeight: "bold", marginBottom: 12 },
  userCard: {
    backgroundColor: "#fff",
    padding: 12,
    marginBottom: 8,
    borderRadius: 8,
    shadowColor: "#000",
    shadowOpacity: 0.1,
    shadowRadius: 4,
    elevation: 2,
  },
  name: { fontSize: 16, fontWeight: "600" },
  email: { fontSize: 14, color: "#666", marginTop: 4 },
});

export default ListDemo;
```

**Tại sao cần `key`?**

- React dùng `key` để nhận biết item nào thay đổi, thêm, hoặc xóa
- `key` phải **duy nhất** trong list
- **Không dùng index** làm key nếu list có thể thay đổi thứ tự

### 2.5 Fragment: `<>...</>` hoặc `<React.Fragment>`

## 3. Component Composition

### 3.1 Parent-Child Relationship

Components tạo thành cây (tree) — component cha chứa component con.

```tsx
import React from "react";
import { View, Text, StyleSheet } from "react-native";

// 👶 Child Component
const Header = () => (
  <View style={styles.header}>
    <Text style={styles.headerText}>My App</Text>
  </View>
);

// 👶 Child Component
const Footer = () => (
  <View style={styles.footer}>
    <Text style={styles.footerText}>© 2026 My App</Text>
  </View>
);

// 👶 Child Component
const Content = () => (
  <View style={styles.content}>
    <Text>Nội dung chính ở đây</Text>
  </View>
);

// 👨 Parent Component — tổ hợp các child components
const AppLayout = () => {
  return (
    <View style={styles.container}>
      <Header />
      <Content />
      <Footer />
    </View>
  );
};

const styles = StyleSheet.create({
  container: { flex: 1 },
  header: { padding: 16, backgroundColor: "#007AFF" },
  headerText: { color: "#fff", fontSize: 20, fontWeight: "bold" },
  content: { flex: 1, padding: 16 },
  footer: { padding: 16, backgroundColor: "#f0f0f0" },
  footerText: { textAlign: "center", color: "#666" },
});

export default AppLayout;
```

### Bài 3: Conditional UI

**Yêu cầu**: Component hiển thị UI khác nhau dựa trên trạng thái.

```tsx
// exercises/ConditionalUI.tsx
import React from "react";
import { View, Text, StyleSheet, ActivityIndicator } from "react-native";

type Status = "loading" | "success" | "error" | "empty";

interface DataDisplayProps {
  status: Status;
  data?: string[];
  errorMessage?: string;
}

const DataDisplay = ({ status, data, errorMessage }: DataDisplayProps) => {
  // Loading state
  if (status === "loading") {
    return (
      <View style={styles.center}>
        <ActivityIndicator size="large" color="#007AFF" />
        <Text style={styles.loadingText}>Đang tải dữ liệu...</Text>
      </View>
    );
  }

  // Error state
  if (status === "error") {
    return (
      <View style={[styles.center, styles.errorBox]}>
        <Text style={styles.errorIcon}>❌</Text>
        <Text style={styles.errorText}>{errorMessage || "Có lỗi xảy ra"}</Text>
      </View>
    );
  }

  // Empty state
  if (status === "empty" || !data || data.length === 0) {
    return (
      <View style={styles.center}>
        <Text style={styles.emptyIcon}>📭</Text>
        <Text style={styles.emptyText}>Không có dữ liệu</Text>
      </View>
    );
  }

  // Success state
  return (
    <View>
      <Text style={styles.successTitle}>✅ Dữ liệu ({data.length} items):</Text>
      {data.map((item, index) => (
        <Text key={index} style={styles.dataItem}>
          • {item}
        </Text>
      ))}
    </View>
  );
};

// Demo tất cả trạng thái
const ConditionalUIExercise = () => {
  return (
    <View style={styles.container}>
      <Text style={styles.heading}>Conditional Rendering Demo</Text>

      <Text style={styles.label}>Loading:</Text>
      <DataDisplay status="loading" />

      <Text style={styles.label}>Error:</Text>
      <DataDisplay status="error" errorMessage="Không thể kết nối server" />
```

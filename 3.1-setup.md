# 3.1 Môi Trường & Setup

> **Mục tiêu**: Cài đặt môi trường phát triển React Native hoàn chỉnh.

---

## 📖 Tổng Quan

Có 2 cách chính để phát triển React Native:

|                      | **Expo** (Recommended)   | **React Native CLI**             |
| -------------------- | ------------------------ | -------------------------------- |
| **Setup**            | 5 phút                   | 30-60 phút                       |
| **Yêu cầu**          | Node.js                  | Node.js + Android Studio + Xcode |
| **Test trên device** | Expo Go app              | Build native riêng               |
| **Native modules**   | Expo SDK (hạn chế)       | Full access                      |
| **Khi nào dùng**     | Học, prototype, 90% apps | Custom native code               |

**👉 Recommend: Bắt đầu với Expo**

## 1. Expo Setup (Recommended)

### 1.1 Cài đặt Node.js

```bash
# Kiểm tra Node.js đã cài chưa
node --version    # Cần >= 18.0.0
npm --version     # Đi kèm Node.js

# Nếu chưa có, cài từ: https://nodejs.org/
# Chọn phiên bản LTS (Long Term Support)

# Hoặc dùng nvm (Node Version Manager) — recommended
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install --lts
nvm use --lts
```

### 1.2 Tạo Project Expo

```bash
# Tạo project mới với TypeScript template
npx create-expo-app@latest MyApp --template blank-typescript

# Di chuyển vào thư mục project
cd MyApp

# Cài dependencies
npm install
```

### 1.3 Chạy App

```bash
# Khởi động development server
npx expo start

# Các options:
# Nhấn 'a' — mở Android emulator
# Nhấn 'i' — mở iOS simulator (chỉ macOS)
# Nhấn 'w' — mở web browser
# Scan QR code — mở trên Expo Go (physical device)
```

### 1.4 Cài Expo Go trên Điện Thoại

1. **Android**: Tìm "Expo Go" trên Google Play Store
2. **iOS**: Tìm "Expo Go" trên App Store
3. Scan QR code từ terminal/browser
4. App sẽ load trực tiếp trên điện thoại!

### 1.5 Chạy trên Emulator/Simulator

**Android Emulator:**

```bash
# 1. Cài Android Studio: https://developer.android.com/studio
# 2. Mở Android Studio → More Actions → Virtual Device Manager
# 3. Tạo device mới (recommend: Pixel 7, API 34)
# 4. Start emulator
# 5. Trong terminal expo, nhấn 'a'
```

**iOS Simulator (macOS only):**

```bash
# 1. Cài Xcode từ Mac App Store
# 2. Mở Xcode → Settings → Platforms → Install iOS Simulator
# 3. Trong terminal expo, nhấn 'i'
```

## 2. React Native CLI (Hiểu Cơ Bản)

### 2.1 Khác biệt Expo vs CLI

```
Expo:
┌──────────────────────────────────┐
│  JavaScript Code (bạn viết)      │
│  ┌────────────────────────────┐  │
│  │  Expo SDK (nhiều APIs)      │  │
│  │  ┌──────────────────────┐  │  │
│  │  │  React Native Core    │  │  │
│  │  └──────────────────────┘  │  │
│  └────────────────────────────┘  │
│  Expo Go runtime (có sẵn)        │
└──────────────────────────────────┘

CLI (Bare):
┌──────────────────────────────────┐
│  JavaScript Code (bạn viết)      │
│  ┌────────────────────────────┐  │
│  │  React Native Core         │  │
│  │  + Native modules (tự cài) │  │
│  └────────────────────────────┘  │
│  Custom native build (tự build)  │
└──────────────────────────────────┘
```

### 2.2 Khi Nào Cần Eject / Dùng CLI

- Cần custom native module (Java/Kotlin, Swift/ObjC)
- Cần library không support Expo (hiếm gặp ngày nay)
- Cần tối ưu bundle size đến mức tối đa
- Project enterprise với yêu cầu đặc biệt

> 💡 **Tip**: Expo hiện đã hỗ trợ **EAS Build** và **Development Build** — gần như không cần eject nữa!

### 2.3 Tạo Project CLI (tham khảo)

```bash
# Chỉ khi thực sự cần CLI
npx react-native init MyApp --template react-native-template-typescript
cd MyApp

# Android
npx react-native run-android

# iOS (macOS only)
npx react-native run-ios
```

## 3. Project Structure

### 3.1 Cấu trúc thư mục Expo project

```
MyApp/
├── .expo/                  # Expo cache (gitignore)
├── assets/                 # Ảnh, fonts, etc.
│   ├── adaptive-icon.png
│   ├── favicon.png
│   ├── icon.png
│   └── splash-icon.png
├── node_modules/           # Dependencies (gitignore)
├── src/                    # 👈 Source code (tự tạo)
│   ├── components/         # Reusable components
│   │   ├── Button.tsx
│   │   └── Card.tsx
│   ├── screens/            # Screen components
│   │   ├── HomeScreen.tsx
│   │   └── ProfileScreen.tsx
│   ├── contexts/           # React Contexts
│   │   └── ThemeContext.tsx
│   ├── hooks/              # Custom hooks
│   │   └── useFetch.ts
│   ├── services/           # API services
│   │   └── api.ts
│   ├── utils/              # Utility functions
│   │   └── helpers.ts
│   ├── constants/          # Constants
│   │   ├── colors.ts
│   │   └── spacing.ts
│   ├── types/              # TypeScript types
│   │   └── index.ts
│   └── navigation/         # Navigation config
│       └── AppNavigator.tsx
├── App.tsx                 # Entry point
├── app.json                # App config
├── babel.config.js         # Babel config
├── package.json            # Dependencies & scripts
└── tsconfig.json           # TypeScript config
```

### 3.2 Các File Config Quan Trọng

**package.json** — Quản lý dependencies và scripts:

```json
{
  "name": "myapp",
  "version": "1.0.0",
  "main": "expo/AppEntry.js",
  "scripts": {
    "start": "expo start",
    "android": "expo start --android",
    "ios": "expo start --ios",
    "web": "expo start --web",
    "lint": "eslint ."
  },
  "dependencies": {
    "expo": "~52.0.0",
    "react": "18.3.1",
    "react-native": "0.76.0"
  },
  "devDependencies": {
    "@types/react": "~18.3.0",
    "typescript": "~5.3.3"
  }
}
```

**app.json** — Cấu hình app (tên, icon, splash, permissions):

```json
{
  "expo": {
    "name": "My App",
    "slug": "my-app",
    "version": "1.0.0",
    "orientation": "portrait",
    "icon": "./assets/icon.png",
    "userInterfaceStyle": "light",
    "newArchEnabled": true,
    "splash": {
      "image": "./assets/splash-icon.png",
      "resizeMode": "contain",
      "backgroundColor": "#ffffff"
    },
    "ios": {
      "supportsTablet": true,
      "bundleIdentifier": "com.yourname.myapp"
    },
    "android": {
      "adaptiveIcon": {
        "foregroundImage": "./assets/adaptive-icon.png",
        "backgroundColor": "#ffffff"
      },
      "package": "com.yourname.myapp"
    },
    "web": {
      "favicon": "./assets/favicon.png"
    }
  }
}
```

**tsconfig.json** — TypeScript configuration:

```json
{
  "extends": "expo/tsconfig.base",
  "compilerOptions": {
    "strict": true,
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  },
  "include": ["**/*.ts", "**/*.tsx"]
}
```

**babel.config.js**:

```js
module.exports = function (api) {
  api.cache(true);
  return {
    presets: ["babel-preset-expo"],
  };
};
```

## 4. Cấu Hình Thường Dùng

### 4.1 Path Aliases (@/)

Thay vì `../../../components/Button`, dùng `@/components/Button`.

```bash
# Cài babel plugin
npm install --save-dev babel-plugin-module-resolver
```

```js
// babel.config.js
module.exports = function (api) {
  api.cache(true);
  return {
    presets: ["babel-preset-expo"],
    plugins: [
      [
        "module-resolver",
        {
          root: ["./"],
          alias: {
            "@": "./src",
          },
        },
      ],
    ],
  };
};
```

```tsx
// Trước:
import Button from "../../../components/Button";

// Sau:
import Button from "@/components/Button";
```

### 4.2 ESLint + Prettier

```bash
# Cài ESLint + Prettier
npm install --save-dev eslint prettier eslint-config-prettier eslint-plugin-react eslint-plugin-react-native @typescript-eslint/eslint-plugin @typescript-eslint/parser
```
